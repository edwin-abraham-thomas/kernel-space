# kernel_space — Project Document

## Executive Summary

**kernel_space** is a video content creation project focused on producing educational engineering content for:
- Working engineers
- People learning to become engineers
- Focus on production-grade AI systems

This document outlines the project vision, scope, and execution plan.

---

## 📊 Project Overview

### Vision
Build a comprehensive video series that bridges the gap between theory and practice in AI systems engineering. Each video should be production-grade, well-researched, and immediately actionable.

### Mission
Create content where viewers "tinker, experiment, and break things to truly understand them."

### Core Values
- **Deep > Surface** — In-depth technical content, not superficial coverage
- **Practical > Theoretical** — Runnable code, real examples, applicable insights
- **Quality > Speed** — Production polish matters; better to be late than low-quality
- **Open > Gatekeeping** — Share learnings, patterns, and hard-won knowledge

### Content Guidelines

**Each video should:**
- Be **5-15 minutes** (focus over filler)
- Include **working code** or **real examples**
- Have **clear production quality** (audio, visuals, editing)
- Feature **specific timestamps** for key sections
- Include **resources/links** in video description on YouTube
- Be **self-contained** (no "watch part 1 first" unless series)

**What we avoid:**
- Fluff or padding to hit time limits
- Unsubstantiated claims
- Overly promotional content
- Outdated tooling without noting that

## 🎬 Production Workflow

### Per-Video Process

1. **Planning Phase**
   - Research topic thoroughly
   - Outline key points
   - Identify visuals/demos needed
   - Create script draft

2. **Script Phase**
   - Write recording script (`script.md`)
   - Add production notes (`production.md`)
   - Review for flow and clarity
   - Time estimate the script

3. **Recording Phase**
   - Record voiceover
   - Capture screen recordings if needed
   - Record any live demos

4. **Production Phase**
   - Edit video (cut, transitions, audio)
   - Add graphics, overlays, B-roll
   - Color grade and audio mix
   - Create thumbnail

5. **Publishing Phase**
   - Upload to YouTube
   - Write description with resources
   - Add chapters and timestamps
   - Promote (social, communities, etc.)

### Repository Workflow

Each video folder contains:
- `script.md` — Recording voiceover (clean, no production notes)
- `production.md` — Technical notes (shots, overlays, timing cues)
- `status.md` — Workflow tracking and blockers

**Example Status Flow:**
```
- [ ] Draft script
- [ ] Record voiceover
- [ ] Edit video
- [ ] Publish
```

## 🎨 Technical Specifications

### Video Format
- **Resolution:** 1080p (1920×1080)
- **Frame Rate:** 60fps (24fps is also acceptable for talking head)
- **Codec:** H.264/AVC
- **Target Duration:** 5-15 minutes per video

### Audio
- **Sample Rate:** 48kHz
- **Bitrate:** 128kbps AAC minimum
- **Quality Focus:** Clear voiceover, no room noise
- **Background Music:** Royalty-free, licensed

### Visuals
- **Branding:** kernel_space logo and banner (in `assets/`)
- **Font:** Clear, readable sans-serif (e.g., Inter, Fira Sans)
- **Color Scheme:** Professional, consistent across videos
- **B-roll:** Screen recordings, code, diagrams
